## How It Works

1. **Analyzes Changes**
   Reads git history to detect affected packages

2. **Determines Version Bump**
   Uses commit messages or branch patterns

3. **Updates Versions**
   Modifies package.json files appropriately

4. **Generates Artifacts**
   Creates changelog, commits, and tags

5. **Ready for Publishing**
   Outputs clean package state for your CI/CD pipeline
